建站

在终端输入命令hexo init <folder>、在本地新建文件夹并生成所需材料。其中，<folder>是文件夹名称，也就是你的站点文件夹根目录。

如何找到文件夹：

  • 全局搜索法：（MacOS）CMD + 空格，搜索文件夹名称即可找到。
  • 终端法：在终端输入cd <folder>，即可将终端路径切换到该文件夹下。

GitHub Pages

注册GitHub账号

创建Repository（GitHub Page）

  • GitHub网站：找到+加号，点击创建Repository，需设置为Public
  • 电脑本地：在终端输入ssh-keygen -t rsa -C your_email@youremail.com，一路回车，最终会在用户文件夹下生成电脑的SSH Key。找到电脑的用户文件夹，CMD + Shift + .，可显示隐藏的文件夹.ssh，找到其中的id_rsa.pub文件，用文件编辑器打开，复制其中的文本（即SSH Key）
  • GitHub网站：点击个人头像，在Settings里找到SSH and GPG Keys，点击New SSH Keys；填写Title，并在Key里粘贴SSH Key，点击Add SSH Keys即可
  • 电脑本地：在终端输入ssh -T git@github.com，看到Are you sure you want to continue connecting (yes/no/[fingerprint])?后输入yes回车，即可看到Hi <your_GitHub_name>! You've successfully authenticated, but GitHub does not provide shell access.

如何将GitHub Repository从Private改为Public：

  1. 在GitHub找到对应仓库，在仓库的Settings里下拉找到Options标签
  2. 下拉找到Danger Zone部分，点击Change repository visibility，逐步确认即可

在电脑本地操作以下步骤：

  • 在终端输入cd <folder>打开本地博客根目录，输入npm install hexo-deployer-git --save安装Git部署插件。

  • 找到~/<folder>/_config.yml站点配置文件并打开，

  • 在终端输入命令hexo clean && hexo g && hexo d（若返回INFO Deploy done: git表示部署成功）

    可能会多次报错fatal: Authentication failed for 'https://github.com/<your_GitHub_name>/<RepositoryName>.github.io.git'，解决方法：在GitHub网站，点击头像找到Settings，在左侧导航栏最下方找到Developer settings；在Personal access tokens找到Token(classic)点击进入，点击Generate new token，勾选token并点击Generate token，即可获取身份令牌（用作git push的password）。

  • 复制https://<RepositoryName>.github.io/到浏览器，即可访问博客！
